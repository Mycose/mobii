\subsection{Detailed Description}
Defines an exact duration of time in milliseconds. 

The implementation of this interface may be mutable or immutable. This interface only gives access to retrieve data, never to change it. 

Methods that are passed a duration as a parameter will treat {\ttfamily null} as a zero length duration. 

The
\begin{DoxyCode}
compareTo 
\end{DoxyCode}
 method is no longer defined in this class in version 2.\-0. Instead, the definition is simply inherited from the
\begin{DoxyCode}
Comparable 
\end{DoxyCode}
 interface. This approach is necessary to preserve binary compatibility. The definition of the comparison is ascending order by millisecond duration. Implementors are recommended to extend
\begin{DoxyCode}
AbstractInstant 
\end{DoxyCode}
 instead of this interface.

Converts this duration to a \hyperlink{classorg_1_1joda_1_1time_1_1_period}{Period} instance using the standard period type and the I\-S\-O chronology. 

Only precise fields in the period type will be used. Thus, only the hour, minute, second and millisecond fields on the period will be used. The year, month, week and day fields will not be populated. 

If the duration is small, less than one day, then this method will perform as you might expect and split the fields evenly. If the duration is larger than one day then all the remaining duration will be stored in the largest available field, hours in this case. 

For example, a duration effectively equal to (365 + 60 + 5) days will be converted to ((365 + 60 + 5) $\ast$ 24) hours by this constructor. 

For more control over the conversion process, you must pair the duration with an instant, see \hyperlink{classorg_1_1joda_1_1time_1_1_period_ad5453618f3627435523b38382dad0ac1}{Period\#\-Period(\-Readable\-Instant,\-Readable\-Duration)}.
